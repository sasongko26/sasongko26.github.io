Could not find in locally installed gem

15 Jun 2025

Ketika akan menjalankan blog secara lokal yang menggunakan jekyll ternyata muncul error.

$ bundle exec jekyll serve
Could not find uri-1.0.2 in locally installed gems
Run `bundle install` to install missing gems.

Kemudian mengikuti saran yang diberikan, masih tetap error.

$ bundle install
Because your Gemfile specifies no global remote source, your bundle is locked to uri (1.0.2) from locally installed gems. However, uri (1.0.2) is not installed. You'll need to either add a global remote source to your Gemfile or make sure uri (1.0.2) is available locally before rerunning
Bundler.

Solusinya:

$ bundle update
Resolving dependencies...
benchmark-0.4.0 is built in to Ruby, and can't be cached because your Gemfile doesn't have any sources that contain it.
uri-1.0.3 is built in to Ruby, and can't be cached because your Gemfile doesn't have any sources that contain it.
Using uri 1.0.3 (was 1.0.2)
securerandom-0.4.1 is built in to Ruby, and can't be cached because your Gemfile doesn't have any sources that contain it.
json-2.9.1 is built in to Ruby, and can't be cached because your Gemfile doesn't have any sources that contain it.
net-http-0.6.0 is built in to Ruby, and can't be cached because your Gemfile doesn't have any sources that contain it.
Bundle updated!

Kemudian run jekyll lagi. Alhamdulillah lancar aman jaya.
